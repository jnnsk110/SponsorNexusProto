## 1. サービス概要 (Concept)

### サービス名
**Sponsor Nexus (スポンサー・ネクサス)**
※仮称。佐藤卓也氏の構想する「平和と文化のスポンサーシッププラットフォーム」

### コンセプト
**「企業の志（CSR/PR）」と「地域の熱量（プロジェクト）」を、"AI"と"人の目利き"で結びつけるプラットフォーム。**
単なる自動マッチングではなく、AIが分析したデータを元に、運営事務局が信頼できる「お見合い（紹介）」をセッティングする。これにより、企業・プロジェクト双方にとって安心で質の高い支援関係を構築する。

## 2. ユーザーロール定義 (User Roles)

| ロール                            | ユーザー像                       | 主な目的                                            | テーマカラー                      |
| :-------------------------------- | :------------------------------- | :-------------------------------------------------- | :-------------------------------- |
| **1. 運営事務局 (Admin)**         | 佐藤卓也氏、事務局メンバー       | 全体の品質管理、AI分析に基づくマッチング仲介        | **Deep Navy** (信頼・統率)        |
| **2. 企業スポンサー (Sponsor)**   | 大企業のCSR担当、広報担当        | 確実でブランディングに資する支援先の発見            | **Corporate Blue** (知性・誠実)   |
| **3. プロジェクトオーナー (Owner)** | 地域の団体代表、イベント主催者   | 資金調達、信頼できる企業とのパートナーシップ        | **Forest Green** (成長・現場)     |

## 3. サービス機能構想 (Service Logic)

本サービスは、単なる送金ツールではなく、**「資金」と「信頼（レポート）」を循環させる3者間のクローズド・エコシステム**として機能します。

### A. 信頼の担保機能（Foundation）
*   **[Admin] 厳格な審査ゲート (Quality Gate)**
    *   **誰が:** 運営事務局が
    *   **誰のために:** 全参加ユーザーのために
    *   **何をする:** 参加する企業・団体を事前審査し、承認されたユーザーのみがログインできる「招待制」を維持する。これにより、オープンなクラファンにはない「顔の見える安心感」を担保する。

### B. 支援のマッチング機能（Matching Logic）
*   **[Admin & AI] "AIの眼" x "人の手" による紹介**
    *   **誰が:** AIが分析し、運営事務局が
    *   **誰のために:** 企業スポンサーのために
    *   **何をする:**
        1.  企業の方針（AI Brain設定）とプロジェクト内容をAIが解析・マッチング。
        2.  事務局がその妥当性を確認し、**「事務局からの推奨」として**企業に紹介リストを届ける。
        3.  企業は「自分で探す」手間なく、信頼できる支援先に出会える。

### C. 価値の還元機能（Reporting Logic）
*   **[Owner & AI] CSR資産の自動生成エコシステム**
    *   **誰が:** 現場（プロジェクト）とAIが
    *   **誰のために:** 企業スポンサーのために
    *   **何をする:**
        1.  現場は「写真と一言メモ」を送るだけ（デジタル負荷ゼロ）。
        2.  AIがそれを**「企業の広報がそのまま使えるCSRレポート記事」**に変換する。
        3.  支援したお金が、「活動報告」という**企業のPR資産**になって返ってくる仕組みを提供する。

### D. 資金と信用の循環機能（Transaction & Evaluation）
*   **[System] 信用スコアリング (Credit Loop)**
    *   **誰が:** プラットフォーム（システム）が
    *   **誰のために:** 次のマッチングのために
    *   **何をする:**
        *   「誠実に報告したか（Owner）」「継続的に支援したか（Sponsor）」をスコア化・可視化する。
        *   スコアが高いユーザーほど、良質なマッチングが優先的に紹介されるインセンティブを働かせる。

## 4. Web画面機能構想 (UI/UX)

実際の画面上に実装される機能と、ユーザー体験の設計です。

### ① 運営事務局画面 (Admin View)
プラットフォームの信頼性と収益性を管理するコックピット機能。

*   **全体ダッシュボード (Dashboard)**
    *   **機能:** プラットフォームの「健全性」を瞬時に把握する。
    *   **KPI:** 「今月の流通総額（GMV）」「新規登録待ち件数」「マッチング成立率」。
    *   **Alert:** 「承認待ちの企業ユーザー」「内容確認が必要なプロジェクト」など、管理者のアクションが必要な項目を赤字で通知。

*   **スポンサー一覧 (Sponsor List)**
    *   **機能:** 支援企業（Funder）の情報を一元管理する。
    *   **管理項目:** 企業基本情報、契約ステータス（プラチナ/ゴールド等）、担当者連絡先。
    *   **AI Brain確認:** 各企業が設定した「AI Brain（支援方針パラメータ）」を閲覧し、どのようなプロジェクトを求めているかを把握・代理設定する機能。

*   **オーナー一覧 (Owner List / Organization Management)**
    *   **機能:** プロジェクトを主催する**団体（Organization）**の審査・管理を行う。
    *   **重要性:** 「プロジェクト（企画）」の前に「主催者（人・法人）」が信頼できるかをチェックする、コンプライアンス上の防波堤。
    *   **管理項目:** 法人格、設立年数、役員情報、反社チェック状況、過去のトラブル有無など。

*   **プロジェクト一覧 (Project List)**
    *   **機能:** 支援を求める**個別の企画（Project）**を審査・管理する。
    *   **審査ステータス:** 「申請中」「審査OK」「活動中」「完了」。
    *   **AI診断結果:** AIによるプロジェクト採点（S/A/B/C）を表示し、内容が不十分なプロジェクトには「写真追加のアドバイス」などを送る。

*   **レポート管理 (Reports)**
    *   **機能:** スポンサーへ届ける活動報告の品質を最終チェックする「関所」。
    *   **承認待ちドラフト (Approval Queue):**
        *   現場が投稿し、AIが生成したレポートの「下書き」を一覧表示する。
        *   **詳細ポップアップ機能:** リストをクリックするとモーダルが開き、レポート概要の確認・修正指示・PDFプレビューが可能。
        *   **アクション:** 内容に問題がなければ「承認して送信」ボタンでスポンサーへ即時送付される。
    *   **送信済み履歴 (Sent History):**
        *   過去に送信した全レポートをリスト化し、いつでもPDFとして再ダウンロードが可能。
        *   「どのスポンサーに、いつ、どんなレポートを送ったか」を証憑として管理する。

*   **入出金管理 (Payments / Escrow)**
    *   **機能:** マッチング成立後の「請求・入金確認・送金」を一元管理する。
    *   **エスクロー:** 事務局が一時的に支援金を預かることで、支払いの安全性とプロジェクトへの確実な資金提供を保証する。
    *   **手数料管理:** プラットフォーム手数料（例: 10%）を差し引いてプロジェクトへ送金する計算を自動化し、ビジネス収益を可視化する。

### ② 企業スポンサー画面 (Sponsor View)
予算を預かる担当者が「説明責任（Accountability）」を果たしつつ、効果的な支援を行うための画面。

*   **企業ダッシュボード (Corporate Dashboard)**
    *   **機能:** 自社の支援状況と成果（ROI）を可視化する。
    *   **KPI:** 「本年度の支援総額」「CSR貢献スコア（AI算出）」「受領レポート数」。
    *   **Action:** 「支援中プロジェクト」をクリックすると、進捗状況やリターン内容の詳細ポップアップが表示され、現状を即座に把握できる。

*   **プロジェクトを探す (Find Projects & Review)**
    *   **機能:** 事務局から推奨された「お墨付き」プロジェクトを閲覧・検討する。
    *   **UI:** **リッチカード形式**で、プロジェクトの魅力的な画像、マッチ度（AIスコア）、推奨理由（Best Matchバッジ等）を視覚的に訴求。
    *   **詳細確認:** カードをクリックすると詳細モーダルが開き、募集金額や主催団体情報をスムーズに確認できる。
    *   **団体プロフィール閲覧:** 詳細モーダル内の団体名をクリックすると、**「団体プロフィール（役員、設立歴、ガバナンス情報）」**がポップアップ表示され、その場で信頼性を確認できる。

*   **支援履歴 (Support History)**
    *   **機能:** 過去の支援実績と、それに紐づく活動レポートをアーカイブする。
    *   **資産化:** 「2025年度活動報告書」のように、年度ごとの支援実績を自動でまとめ、企業の統合報告書などに使える素材としてダウンロードできる。

*   **企業設定 (Corporate Settings / AI Brain)**
    *   **機能:** AIマッチングの精度を高めるための「企業の意思」を入力・管理する。
    *   **デザイン:** プレミアムなUIで、スライダーによる価値バランス調整（CSR重視 vs PR重視）や、注力領域タグの選択を直感的に行える。

### ③ プロジェクトオーナー画面 (Owner View)
活動の意義を伝え、資金を獲得し、説明責任を果たすための現場向けツール。

*   **マイプロジェクト管理 (My Projects)**
    *   **機能:** 自団体が運営する**複数のプロジェクトを一元管理**するダッシュボード。
    *   **管理項目:**
        *   **マルチプロジェクト:** 「募集中」「審査中」「終了分」をタブやリストで整理し、団体としての活動ポートフォリオを可視化する。
        *   **ステータス管理:** 各プロジェクトの調達状況や残り日数を進捗バーで直感的に把握可能。

*   **オファー受信箱 (Offer Box)**
    *   **機能:** 企業からの支援オファーを確認・承諾する。
    *   **UI:** **詳細カード形式**を採用。企業ロゴ・提案金額・条件（デジタルサイネージ掲出など）を大きく表示。
    *   **アクション:** どのプロジェクトに対するオファーかも明記され、「NEW」バッジで新着を見逃さない。

*   **活動報告の提出 (Easy Reporting)**
    *   **機能:** 現場の負担を極限まで減らした報告ツール。
    *   **マルチ対応:** **報告対象のプロジェクトを選択**し、活動写真をアップロードするだけで紐づけが完了する。
    *   **AI生成:** AIがメモと写真を元に、スポンサー企業の広報担当者が喜ぶレベルの「公式レポート」を自動生成する。

*   **団体設定 (Organization Settings / Governance)**
    *   **機能:** 団体の信頼性を担保するための詳細なプロフィール情報を設定する。**企業の審査基準（コンプライアンス）を満たすための重要機能。**
    *   **詳細項目:**
        *   **基本情報:** 団体名称、法人格（NPO/社団等）、**設立年月・活動歴**。
        *   **ビジョン:** 解決したい社会課題（Mission）、活動内容（Description）。
        *   **ガバナンス:** **代表者名・役員人数**、**情報公開状況**（定款・決算書の有無チェック）、受賞歴・メディア掲載実績。

## 5. 業務フロー (Workflow & Money Flow)

1.  **Entry & Vetting (審査・登録):**
    *   企業・プロジェクトが登録申請。
    *   **事務局が厳格な審査（反社チェック・実態確認）を行い、IDを発行する。**
2.  **Analysis (AI分析):**
    *   AIが双方のデータを解析し、ベストマッチ候補を抽出。企業の「AI Brain」に提案する。
3.  **Introduction (紹介・オファー):**
    *   事務局が紹介内容を確認し、ボタン一つでプロジェクトへオファー（紹介状）を送付。
4.  **Agreement & Escrow (合意・仮払い):**
    *   プロジェクトがオファーを承諾（マッチング成立）。
    *   **企業は支援金を決済し、事務局が一時預かり（エスクロー）する。**
5.  **Activity & Report (活動・報告):**
    *   プロジェクトが活動し、写真をアップロード。AIがレポートを作成し、事務局が承認・送付。
6.  **Payment & Evaluation (支払い・評価):**
    *   レポート受領後、**事務局からプロジェクトへ支援金が入金される。**
    *   企業はレポートを評価し、その結果がプロジェクトの「信用スコア」に加算される（次回のAI推奨時に有利になる）。

## 6. 技術スタック案 (Tech Stack Roadmap)

現状の「張りぼて」から、コストを抑えつつ段階的にシステム化するロードマップ案。

### Phase 0: Prototype (現在の状態)
*   **Type:** 画面モックアップ (Look & Feel Verification)
*   **Tech:** HTML / CSS / Vanilla JS
*   **Infra:** GitHub Pages (Static Hosting)
*   **Data:** なし（ハードコードされたダミーデータ）
*   **目的:** 佐藤様や初期の協力者に「どんな世界観か」を視覚的に伝え、フィードバックを得る。

### Phase 1: MVP (Concierge Model / Human-in-the-Loop)
*   **Type:** コンシェルジュ型MVP (Market Validation)
*   **Tech:** No-Code / Low-Code Tools
    *   **Frontend:** Studio or Webflow (LP + CMS)
    *   **Backend:** Google Sheets / Airtable (データベース代わり)
    *   **Form:** Tally / Typeform (申請受付)
    *   **Workflow:** Dify (AI処理 + メール送信などの外部連携を一元管理)
*   **AI:** Gemini 2.0 Flash API (高速・低コストなモデルでレポート生成)
*   **目的:** **「システム開発費」を極限まで抑え**、事務局の手作業（スプレッドシート管理）中心で実際にマッチングと集金を回し、ビジネスとして成立するか検証する。

### Phase 2: Systematization (Web Application)
*   **Type:** 完全なWebアプリケーション (Scaling)
*   **Tech:** Modern Web Stack
    *   **Frontend:** Next.js (React) + Tailwind CSS
    *   **Backend:** Supabase (Auth, Database, Edge Functions)
    *   **Payment:** Stripe Connect (プラットフォーム決済)
*   **AI:** RAG (Retrieval-Augmented Generation) を構築し、企業の過去の傾向学習や、より高度なマッチング精度を実現。
*   **目的:** ユーザー数が100を超え、手作業の限界が来た段階で移行。自動化により運用コストを下げる。
